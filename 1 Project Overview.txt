1) Project Overview

1.1 Website ka Maqsad

Aik single website jahan user:
	•	Pick & Drop services book kar sake
	•	Ziyarat packages select + inquiry/booking kar sake
	•	Khajoor (dates) services (order/inquiry) kar sake
	•	Book Store (e-commerce via Salla) se books buy kar sake
	•	Ramadan Donations / Sadaqah donate kar sake
	•	Salla Store integration se payment collection + order management ho

1.2 Target Audience
	•	Saudi/GCC visitors + Pakistan visitors
	•	Ziyarat karne wale log, families
	•	Khajoor buyers (gift/wholesale)
	•	Islamic books readers
	•	Donation donors (Ramadan campaigns)

⸻

2) Core Modules (Features)

Module A — Pick & Drop Booking

Purpose: Transport booking (airport/hotel/ziyarat pickup drop)

User Flow
	1.	User “Pick & Drop” page open kare
	2.	Form fill kare:
	•	Pickup location (Google map link / text)
	•	Drop location
	•	Date & time
	•	Passengers count
	•	Luggage (yes/no)
	•	Vehicle type (Sedan/Van/Hiace) (optional)
	•	Contact: Name, Mobile, WhatsApp
	3.	Submit → “Booking Request Received” message
	4.	Admin ko notification (Email/WhatsApp optional)
	5.	Admin status set kare: Pending / Confirmed / Completed / Cancelled

Developer Requirements
	•	Booking form + database saving
	•	Admin panel/listing
	•	Auto confirmation email/SMS optional
	•	WhatsApp “Click to chat” button: prefilled message with booking details

⸻

Module B — Ziyarat Packages

Purpose: Packages listing + booking/inquiry

Package Fields (Admin add kare)
	•	Package name (e.g., Makkah Ziyarat – Half Day)
	•	City (Makkah/Madinah/Taif/Badr etc.)
	•	Duration (hours/days)
	•	Price (per person / per group) + currency
	•	Includes (transport, guide, refreshments)
	•	Excludes
	•	Itinerary points (bullet list)
	•	Pickup point rules
	•	Images (gallery)
	•	Availability notes

User Flow
	1.	User Packages page → package open kare
	2.	“Book Now / Inquiry” button
	3.	Form:
	•	Package selected (auto)
	•	Date
	•	Persons
	•	Pickup location
	•	Contact details
	4.	Submit → admin notification

Optional Upgrade
	•	Online payment (advance payment) via Salla “payment link / product” approach (agar aap chahen)

⸻

Module C — Khajoor Services

Purpose: Dates (khajoor) selling + services (gift packing, bulk, delivery)

Options
	•	Option 1 (Recommended): Khajoor products Salla store me (inventory, payment, shipping Salla handle kare)
	•	Website par: “Khajoor” page with categories + “Buy on Store” buttons that open Salla product pages

Content Sections
	•	Categories (Ajwa, Safawi, Mabroom etc.)
	•	Bulk order inquiry form
	•	Gift box inquiry
	•	Delivery cities supported

⸻

Module D — Book Store (Salla E-commerce)

Purpose: Books selling + payment collection

Integration Approaches (Developer choose kare)
	•	Approach 1 (Simple & Fast): Website menu me “Shop” → Salla store link (subdomain: store.yourdomain.com OR salla URL)
	•	Approach 2 (Semi-Embedded): Selected products “Buy Now” buttons on website → Salla product/cart
	•	Approach 3 (Advanced): Headless style (only if Salla APIs available & budget higher)

Recommended: Approach 1 + Approach 2 (fast, stable, low cost)

⸻

Module E — Donations (Ramadan / Sadaqah)

Purpose: Donation collection with clear purpose & transparency

Donation Types (Website)
	•	Sadaqah
	•	Zakat (optional)
	•	Ramadan Iftar sponsorship (optional)
	•	“General Donation”

User Flow
	1.	Donation page → amount select (50/100/200/custom)
	2.	Donor details (optional): name, phone, note/dua request
	3.	Payment method:
	•	Option A (Recommended): Salla donation product (amount variants) OR payment link
	•	Option B: External payment gateway (if available)
	4.	Success page + receipt message

Admin Needs
	•	Donation records export (CSV)
	•	“Campaign progress” counter (manual/auto)
